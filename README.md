Atomic Design:

Atomic design is a methodology for web and application development in which we emphasize the modularity of the design where we will try to create a design system rather than a page where we take cues from chemistry as it composed of five distinct stages working together to create interface and the five stages are
    1.Atoms
    2.Molecules
    3.Organisms
    4.Templates
    5.Pages
By following this methodology we can create a system which is maintainable as it should stand the test of time.

The 12 factor app:

It is a kind of methodology or a document synthesizes the experience and observations on a wide variety of software-as-a-service apps which give users to insights on some systemic problems that are being experienced in the modern application development.

the tweleve factors are:
1) Codebase = A codebase is any single repo or any set of repos who share a root commit

2) Dependencies = A 12 factor app never relies on implicit existence of system-wide packages

3) Conif = it stores config in environment variables

4) Backing services = It makes no distinction between local and third party services

5) Build, release, run = It uses strick separation between the build, release, and run stages

6) Processes = The processes are stateless and share-nothing

7) Port binding = These are completely self-contained

8) Concurrency = The processes are a first class citizen

9) Disposability = They can be started or stopped at a short notice.

10) Dev/prod parity = keep dev, staging and production as similar as possible this minimize the risk of bugs

11) Logs = Treat logs as events. The logs should be handled permanently regardless of the situation

12) Admin process = management tasks should be run in same environment to ensure consistency.

This is a result of long curated work that is being analyzed and transformed into this kind og guide helping a lot of developers around the globe.
